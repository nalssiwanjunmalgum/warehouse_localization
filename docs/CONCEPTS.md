이 도메인(ROS2 / 로컬라이제이션)이 처음인 관점에서, 나눈 개념 질문과 설명을 모아둡니다.

## RViz의 빨간 점 = LiDAR 스캔(`/scan`)
2D 라이다가 360°를 1°씩 회전하며 빔 360개를 쏘고, 각 빔은 그 방향 **가장 가까운 물체까지 거리**를 잰다(최대 12m).
- **빨간 점 1개** = 빔이 벽/기둥/랙 표면에 맞은 지점
- **이어진 선** = 물체 윤곽(벽=직선, 기둥=작은 점뭉치)
- **점 없음** = 그 방향 12m 안에 아무것도 없음(빔 = inf → 안 찍힘)

즉 빨간 점 = "로봇이 지금 실제로 보고 있는 주변 물체의 실루엣".

**왜 핵심인가**: 로컬라이제이션(AMCL)은 이 현재 스캔을 미리 만든 지도에 겹쳐 맞춰 "내가 지도상 어디인지"를 추정한다.
- 벽/기둥 근처 → 점 풍부 → 맞출 특징 많음 → 위치 추정 잘 됨
- 중앙 개활지 → 점 사라짐 → 맞출 게 없음 → **위치 보정 불가** = 재현하려는 문제(featureless core)

관련: [M2_WORLD_DESIGN.md](./M2_WORLD_DESIGN.md)의 "215빔 → 14빔 붕괴"가 RViz에선 빨간 점이 사라지는 모습.

## 맵(map) vs 궤적(trajectory) — 헷갈리기 쉬움
서로 다른 것이다.

| | 맵 (지도) | 궤적 (trajectory) |
|---|---|---|
| 무엇 | 환경을 위에서 본 2D 도면(벽·기둥 위치) | 로봇이 지나온 경로(시간별 위치) |
| 성격 | 정적(환경 자체) | 이동 이력 |
| 용도 | AMCL이 **위치 추정** 시 대조하는 기준 | 나중에 **정확도 평가**(오차 측정) |

- **맵 = 점유 격자(occupancy grid)**: 각 칸이 검정(벽)/흰색(빈 공간)/회색(모름).
  비유 = 종이 평면도. AMCL은 현재 라이다 스캔(빨간 점)을 이 평면도에 겹쳐 맞춰 위치를 안다.
- **"맵 전략"(P2) 결정** = 이 평면도를 **어떻게 만들지**:
  - (A) SLAM으로 자동 생성(`slam_toolbox`, 로봇 한 바퀴 돌림) — 실제 로봇 방식
  - (B) 월드를 아니까 수동으로 이미지(pgm) 직접 제작
  - ★ 반전: 개활지라 맵 대부분이 흰색(빈 공간) → 지도 있어도 개활 구간엔 대조할 특징 없음 = 문제의 본질.
- **궤적**은 맵이 아니라 **평가(M4)** 재료: 추정 궤적 vs 실제(Gazebo가 아는 정답) 궤적을 비교 → 오차 ATE/RPE 산출.

## odom / base_footprint / TF 프레임 (요약)
- **base_footprint / base_link**: 로봇 자신 기준 좌표.
- **odom 프레임**: 바퀴 회전을 적산한 "출발점 기준 추정 위치". 시간이 지나면 미끄러짐 등으로 **드리프트**(누적 오차). 우리 프로젝트가 잡으려는 문제의 핵심.
- **TF**: 이 좌표계들 사이의 변환 관계. RViz에서 프레임 축으로 표시됨.
- 참고: 본 sim의 waffle는 odom을 스폰 world 좌표(초기 -24,-24)에서 시작해 적산 → `odom.position ≈ world`(드리프트 포함).
