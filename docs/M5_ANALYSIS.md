# M5 분석 — ④ 노이즈 주입 & +EKF 융합 결과/해설

> 목적: 바퀴 오도메트리에 드리프트를 주입(④)하고 `robot_localization` EKF 로 오도+IMU 를 융합했을 때
> **각 수치가 무엇을 뜻하는지** 해설하고, C1~C5 를 baseline 과 비교 분석한다. (3단계 비교의 **2단계=완화**)
> 설계·구성: [M5_EKF.md](./M5_EKF.md) · baseline: [M3_ANALYSIS.md](./M3_ANALYSIS.md) · 원자료: `outputs/ekf/`

## 0. 실험 구성 (baseline 에서 무엇이 바뀌었나)

**환경·맵·시나리오(C1~C5)·시작포즈는 M3 와 100% 동일**([M3_ANALYSIS §0](./M3_ANALYSIS.md) 참조 — 60×60 개활
창고, 20m 격자 기둥 8개, 중앙 featureless core, 12m LiDAR). 바뀐 것은 **오도메트리 처리 한 가지**뿐이다.

### 0-1. 세 가지 구성 — 오도메트리만 다르다
노이즈는 seed 고정(재현성 ⑥)이라 N·E 는 **똑같은 드리프트**를 받는다. 오직 EKF on/off 만 차이.

| 구성 | 오도메트리 | odom→base_footprint TF 발행 | 역할 |
|------|-----------|-----------------------------|------|
| **Baseline** | 완벽(diff_drive, 슬립 0) | diff_drive 플러그인 | M3 — 무노이즈, aliasing 실패만 |
| **N (노이즈만)** | 드리프트 주입 | **noise_injector** | 추측항법이 망가짐 (완화 없음) |
| **E (노이즈+EKF)** | 드리프트 주입 | **EKF**(오도+IMU 융합) | IMU 헤딩으로 드리프트 완화 |

**핵심 통찰 — 노이즈는 "토픽"이 아니라 "TF"에 실려야 한다.** AMCL 의 모션모델은 `/odom` 토픽이 아니라
`odom→base_footprint` **TF** 를 읽어 추측항법을 한다. 그래서 노이즈만 `/odom` 토픽에 실으면 AMCL 은
멀쩡하다. → diff_drive 의 TF 발행을 끈 모델 변형(`warehouse_waffle_ekf`)을 쓰고, 그 자리를
noise_injector(N) 또는 EKF(E)가 채운다.

```
Baseline : map --(AMCL)--> odom --(diff_drive, 완벽)--> base_footprint     드리프트 0
N        : map --(AMCL)--> odom --(noise_injector, TF)--> base_footprint   드리프트 O, 완화 X
E        : map --(AMCL)--> odom --(EKF: 오도속도+IMU yaw)--> base_footprint  드리프트 O, IMU가 헤딩 완화
           (E 에선 noise_injector 는 /odom_noisy 토픽만 내고 TF 는 EKF 담당)
```
> AMCL 은 세 구성 모두에서 그대로 돈다. EKF 는 AMCL 을 **대체하지 않고** 그 밑의 `odom→base` 추측항법만
> 개선한다. 최종 위치추정은 여전히 `/amcl_pose`. → 지표 파이프라인은 M3 와 완전 동일 재사용.

### 0-2. 노이즈 모델 (noise_injector)
깨끗한 속도 `(v, ω)` 에 오차를 실어 재적분한다. 상세·수식: [M5_EKF §2](./M5_EKF.md).

| 성분 | 무엇 | EKF 가 잡나 |
|------|------|------------|
| 병진 스케일 바이어스 `k_v` | 바퀴 지름 오차 등 계통 오차 | 부분적 |
| 병진 랜덤 `σ_v` | 노면 슬립 | 평활만 |
| **회전 바이어스 `b_ω`** ★ | 좌우 바퀴 미세 불일치 → **헤딩이 서서히 틀어짐**(경로가 휨) | **예 — IMU 절대 yaw** |
| 회전 랜덤 `σ_ω` | 회전 슬립 | 부분적 |

`b_ω`(헤딩 바이어스)가 드리프트의 **킬러**다. Gazebo IMU 의 절대 yaw 는 사실상 정확 → EKF 가 이를 융합해
헤딩을 앵커링, `b_ω` 를 상쇄한다. **이것이 EKF 완화의 핵심 원리.** (사용 파라미터 seed=0 고정.)

## 1. 지표 해설 (M5 에서 어떻게 읽나)

지표 정의는 M3 와 동일([M3_ANALYSIS §1](./M3_ANALYSIS.md) 참조: ATE·RPE·drift_rate·cov_major·loop_disp 등).
M5 에서는 지표를 **두 부류로 나눠 읽는 게 핵심**이다. EKF 가 겨냥하는 것과 못 건드리는 것이 다르기 때문:

| 부류 | 지표 | 무엇을 반영 | EKF 로 변하나 |
|------|------|------------|--------------|
| **A. 국소·추측항법 드리프트** | `RPE_1s`, `drift_rate` | 프레임간 얼마나 튀나 / 보정없이 밀리는 속도 | **예 — EKF 가 직접 개선** |
| **B. 절대 위치 확신도** | `cov_major` | AMCL 이 절대 위치를 얼마나 확신하나 | **아니오 — aliasing 은 못 고침** |
| (혼합) | `ATE`, `final_err` | 전체 절대 정확도 | 둘 다 섞여 **aliasing 점프에 지배당함** |

> 즉 EKF 의 효과는 **A(국소 드리프트)에서 봐야** 하고, **B(cov)가 안 변하는 것**이 "aliasing 미해결"의
> 직접 증거다. ATE 는 A·B 가 섞인 데다 aliasing 점프(확률적)에 크게 흔들려 EKF 판정용으론 노이즈가 크다.

### 시나리오별 해석
- **C4 (core 종착) — 결정적 승리 사례.** 노이즈가 baseline 2.39m 를 **10.54m 로 망가뜨렸는데, EKF 가
  2.27m(baseline 동일)로 완전 복구**. RPE 3.35→0.22, drift 0.23→0.00. 짧게 다녀와 aliasing 점프가
  없어 **오차가 온전히 드리프트 탓** → EKF 가 그걸 그대로 걷어냄. **"EKF 는 진짜로 작동한다"의 증명.**
- **C1 (대각 횡단) — 국소는 개선, 절대는 aliasing 이 지배.** RPE **4.38→2.16 (−51%)**, drift 0.58→0.50
  으로 EKF 가 국소 드리프트를 확실히 줄였다. 그런데 final 은 31→47m 로 나빠짐 = 종료 직전 **aliasing
  점프**(EKF 무관, 곡선 그림 참조). ATE 가 EKF 판정에 부적합함을 보여주는 전형.
- **C2 (오프셋 슬라이스)** — ATE 31.9→27.7, drift 0.49→0.38 로 E 가 나음. 단 cov_max 는 최고(31.8m) —
  절대 확신도는 오히려 더 튐. 개선은 국소 드리프트 쪽.
- **C3 (폐루프)** — N·E 거의 동일(35→33). aliasing 이 완전히 지배해 EKF 여지가 거의 없음. cov 는 셋 다 큼.
- **C5 (지그재그) — 나쁜 쪽 사례.** E(31.4)가 N(17.2)보다 **더 나쁨**. EKF 가 아니라 **하필 E 런에서
  aliasing 점프가 더 크게 터짐**(확률적). drift·RPE 도 소폭 악화. → 단일 런 ATE 의 변동성을 그대로 노출.

### 관통 패턴
1. **국소 지표(RPE·drift)는 대체로 EKF 로 개선** (RPE: C1·C4 크게, C2·C3 소폭; drift: C1·C2·C4 개선).
   = EKF 가 실제로 하는 일 = 추측항법 드리프트 완화.
2. **cov_major 는 세 구성 모두 비슷하게 크다**(대부분 6~25m). AMCL 확신도 = 절대위치 실패의 척도인데
   EKF 로 전혀 안 나아진다 → **절대 위치 문제는 손도 못 댔다는 직접 증거.**
3. **ATE·final 은 aliasing 점프(확률적)에 지배**당해 E 가 N 보다 좋기도(C2·C4) 나쁘기도(C1·C5) 하다.
   개활 체류가 길수록(C1·C3·C5) aliasing 이 EKF 신호를 삼킨다. C4(짧음)만 드리프트가 지배해 EKF 가 빛난다.

## 3. 메커니즘 — EKF 가 무엇을 고치고 못 고치나

**두 종류의 오차를 분리해야 한다.**

1. **추측항법 드리프트 (EKF 가 고침).** 회전 바이어스 `b_ω` 로 헤딩이 서서히 틀어지면 경로가 휘어 오차가
   쌓인다. IMU 절대 yaw(정확)를 EKF 가 융합해 헤딩을 앵커링 → 상쇄. **드리프트가 지배하는 국면
   (C4 전체, 모든 시나리오의 초반 상승기)에서 EKF 가 명확히 이긴다.**
2. **AMCL perceptual aliasing (EKF 가 못 고침).** 개활지에서 AMCL 은 똑같이 생긴 기둥에 오매칭되어
   `map→odom` 이 20~40m 씩 **점프**한다. EKF 는 `odom→base`(상대 추측항법)만 개선할 뿐 `map→odom`
   (절대 위치)엔 손을 못 댄다. 점프가 나면 오차는 헤딩 품질과 무관하게 폭발한다.

**두 견고한 결론** (단일 런에서도 흔들리지 않음):
- **cov_major 가 B/N/E 모두 크게 유지** = 절대위치 실패는 EKF 로 전혀 개선 안 됨.
- **C4 의 완전 복구** = 드리프트만 있고 aliasing 이 약하면 EKF 가 baseline 수준으로 회복. 뒤집으면
  나머지 시나리오의 큰 오차는 **드리프트가 아니라 aliasing 탓**임을 증명.

→ **EKF 는 완화(mitigation)이지 해결(solution)이 아니다.** "고칠 수 있는" 추측항법 드리프트는 줄이지만,
개활지의 근본 문제인 **perceptual aliasing(절대 위치 부재)** 은 센서 융합으로 못 고친다.

## 4. 한계 & 다음

**한계(해석 시 유의)**
- **단일 런**: ATE 는 AMCL 파티클필터의 확률적 aliasing 점프에 크게 흔들린다(같은 노이즈라도 AMCL 내부
  난수는 런마다 다름 — C5 가 그 예). 풀 런 ATE 로 EKF 를 엄밀 판정하려면 **반복 평균**(⑥)이 필요.
  견고한 결론(cov 불변 · C4 복구 · 국소지표 개선)은 단일 런에서도 유효.
- **병진 스케일 바이어스**(k_v)는 IMU 로 못 잡아 잔여 드리프트가 남는다(EKF 는 헤딩에 강함).
- 절대 수치는 이 특정 노이즈·환경에서의 값. 중요한 건 **상대 비교와 경향**.

**다음 단계 (§3-B → M6 랜드마크)**
| 값 | baseline | N (노이즈) | E (+EKF) | M6 (+랜드마크) 목표 |
|----|----------|-----------|----------|--------------------|
| 국소 드리프트(RPE·drift) | 中 | 큼 | **완화** | 보정 사이 안정(EKF 유지) |
| 절대 확신도(cov_major) | 폭발 | 폭발 | **여전히 폭발** | **반사판 절대관측으로 급감** |
| aliasing 점프 | 있음 | 있음 | **여전히 있음** | **고유 반사판 삼각측량으로 제거** |

**핵심**: M5 는 "센서 융합만으로는 개활지 aliasing 을 못 이긴다 → 인공 특징(랜드마크)이 필요하다"를
정량 확립했다. EKF 는 M6 에서도 **기반으로 유지**(반사판 보정 사이 추측항법 안정화). M6 는 동일
시나리오·지표로 돌려 **cov_major 와 aliasing 점프가 사라지는지**를 보이면 프로젝트의 결론이 완성된다.
