# M5 분석 — +EKF 센서 융합 결과 (완화 단계)

> 3단계 비교의 **2단계** 결과. 바퀴 오도메트리에 드리프트를 주입(④)하고 `robot_localization` EKF로
> 오도+IMU를 융합했을 때, 드리프트가 얼마나 완화되는지를 C1~C5에서 정량화한다.
> 설계·아키텍처: [M5_EKF.md](M5_EKF.md) · baseline: [M3_ANALYSIS.md](M3_ANALYSIS.md)

---

## 0. 무엇을 비교하나 (3개 구성)

같은 월드·맵·시나리오(C1~C5)·시작포즈에서 **오도메트리 처리만** 바꾼다. 노이즈는 seed 고정(재현성 ⑥).

| 구성 | 오도메트리 | odom→base TF 발행 | 역할 |
|------|-----------|------------------|------|
| **Baseline** | 완벽(diff_drive) | diff_drive | M3, 무노이즈 — aliasing 실패만 |
| **N (노이즈만)** | 드리프트 주입 | noise_injector | 노이즈가 추측항법을 망침 (완화 없음) |
| **E (노이즈+EKF)** | 드리프트 주입 | **EKF**(오도+IMU 융합) | IMU 헤딩으로 드리프트 완화 |

> 세 구성 모두 AMCL 로 최종 위치추정(`/amcl_pose`). EKF 는 AMCL 을 대체하지 않고 그 밑의
> `odom→base` 추측항법만 개선한다. → 지표 파이프라인은 M3 와 완전 동일하게 재사용.

**노이즈 모델(요약, 상세 M5_EKF §2):** 깨끗한 속도 `(v, ω)` 에 스케일 바이어스·랜덤 + **회전 상수
바이어스 `b_ω`(이동 중 헤딩이 서서히 틀어지는 킬러)** 를 실어 재적분. IMU 절대 yaw 가 이 헤딩
바이어스를 상쇄하는 것이 EKF 완화의 핵심.

---

## 1. 지표 (M3 와 동일)

`ATE(RMS)` 전체 정확도 · `drift_rate` 초당 오차증가 · `RPE_1s` 구간 드리프트 ·
`final_err` 종료 시 오차 · `cov_major` AMCL 확신도(1σ 타원 장축). 정의는 [M3_ANALYSIS §1](M3_ANALYSIS.md) 참조.

M5 에서 특히 중요한 두 축:
- **drift_rate / final_err** — EKF 가 직접 겨냥하는 값(추측항법 드리프트). 여기서 개선이 나와야 한다.
- **cov_major** — AMCL 확신도. EKF 로도 **개활지 aliasing 은 못 고치므로** 큰 변화가 없어야 정상
  (= 완화이지 해결 아님).

## 2. 결과

### 전체 C1~C5 (풀 시나리오, 단일 런)

| SCN | Baseline ATE | N ATE | E ATE | drift N→E | 평균 cov (B/N/E) |
|-----|------|------|------|------|------|
| C1 | 34.18 | 27.96 | 29.37 | 0.58→0.50 | 6.8 / 9.4 / 8.4 |
| C2 | 23.95 | 31.92 | **27.65** | 0.49→0.38 | 13.8 / 12.8 / 14.2 |
| C3 | 32.39 | 35.35 | **33.13** | 0.56→0.59 | 9.3 / 5.2 / 6.0 |
| **C4** | 2.39 | 10.54 | **2.27** | 0.23→**0.00** | 6.5 / 7.6 / 6.6 |
| C5 | 26.98 | 17.18 | 31.44 | 0.27→0.37 | 9.5 / 10.4 / 9.4 |

### 읽는 법 — 결과는 "혼합"이며, 그게 핵심이다
풀 개활 횡단(C1·C3·C5)에서 EKF 효과는 **묻힌다**. ATE 가 런마다 25~35m 로 크고 E가 N보다 나은
경우(C2)도 나쁜 경우(C1·C5)도 있다. 반면 **C4 는 명확하다**: 노이즈로 10.54m 로 망가진 ATE 를
EKF 가 **2.27m(baseline 2.39와 동일)로 완전 복구**하고 drift 를 0 으로 만든다.
곡선 그림에서도 C4 만 초록이 시종 낮고, 나머지는 초록·빨강이 40~60m 로 함께 폭주한다.

**왜 이렇게 갈리나 →** §3.

### 사전 검증(스모크, C1 70s 부분 주행)도 같은 이야기
aliasing 대재앙이 터지기 **전** 구간만 보면(동일 seed, EKF on/off): ATE 5.78→2.99m(**−48%**),
final_err 17.0→3.3m(**−81%**), drift 0.73→0.21(**−71%**). 즉 **드리프트가 지배하는 국면에선
EKF 가 크게 이긴다** — 그 국면이 짧아 풀 런 ATE 에선 안 보일 뿐.

> ⚠️ **주의**: 각 구성 **단일 런**이라 ATE 는 AMCL 파티클필터의 확률적 aliasing 점프에 크게 흔들린다
> (같은 노이즈라도 AMCL 내부 난수는 런마다 다름). 풀 런 ATE 로 EKF 를 판정하려면 반복 평균(⑥)이 필요.
> 반면 아래 두 결론은 단일 런에서도 **견고**하다.

## 3. 메커니즘 — EKF 가 무엇을 고치고, 무엇을 못 고치나

**두 종류의 오차를 분리해야 한다:**

1. **추측항법 드리프트 (EKF 가 고침).** 회전 바이어스 `b_ω` 로 헤딩이 서서히 틀어지면 경로가 휘어
   오차가 쌓인다. Gazebo IMU 의 절대 yaw 는 사실상 정확 → EKF 가 융합해 헤딩을 앵커링, 드리프트를
   상쇄. **드리프트가 지배하는 국면(C4 전체, 모든 시나리오의 초반 상승기)에서 EKF 가 명확히 이긴다.**
2. **AMCL aliasing 대재앙 (EKF 가 못 고침).** 개활지에서 AMCL 은 똑같이 생긴 기둥에 오매칭되어
   `map→odom` 이 20~40m 씩 점프한다. EKF 는 `odom→base`(상대 추측항법)만 개선할 뿐 `map→odom`
   (절대 위치)에는 손을 못 댄다. → 점프가 나면 오차는 헤딩 품질과 무관하게 폭발한다.

**두 견고한 결론:**
- **cov_major 는 세 구성(B/N/E) 모두 비슷하게 크다** (표의 마지막 열). AMCL 확신도 = 절대위치
  실패의 척도이고, EKF 로도 전혀 나아지지 않는다. **절대 위치 문제는 손도 못 댔다는 직접 증거.**
- **C4 의 완전 복구** = 드리프트만 있고 aliasing 이 약한 국면에선 EKF 가 baseline 수준으로 회복.
  뒤집으면, 나머지 시나리오에서 오차가 큰 건 **드리프트가 아니라 aliasing 탓**임을 증명.

**결론: EKF 는 완화(mitigation)이지 해결(solution)이 아니다.** 추측항법 드리프트라는 "고칠 수 있는"
오차는 줄이지만, 개활지의 근본 문제인 **perceptual aliasing(절대 위치 부재)** 은 센서 융합으로 못 고친다.
→ 개활지에서 절대 위치를 확정하려면 **인공 특징(랜드마크)** 이 필요하다 → **M6**.

## 4. 한계 & 다음

- EKF 는 헤딩 드리프트에 강하지만 **병진 스케일 바이어스**(k_v)는 IMU 로 못 잡아 잔여 드리프트가 남는다.
- 개활 core 구간에선 세 구성 모두 절대 위치 확신(cov) 이 낮다 = 공통 미해결 과제.
- **M6 (+랜드마크):** 반사판 intensity 검출 + 삼각측량으로 `map→base` 절대 보정 → aliasing 제거,
  cov 급감 목표. M5 의 EKF 는 M6 에서도 기반으로 유지(보정 사이 추측항법 안정화).
