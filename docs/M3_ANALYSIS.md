|------|------|------|

```
   60m × 60m 실내 (위=북/+y, 아래=남/-y, 오른=동/+x)
   ┌──────────────────────────────────────┐  y=+30 (북벽)
   │  ●(-20,20)     ●(0,20)     ●(20,20)  │
   │            ▓(0,16)                    │
   │  ●(-20,0)   ▓(-16,0) ★ ▓(16,0)  ●(20,0) │  ← ★=core(0,0)
   │            ▓(0,-16)                   │
   │  ●(-20,-20)    ●(0,-20)    ●(20,-20)  │
   └──────────────────────────────────────┘  y=-30 (남벽)
     x=-30(서벽)                    x=+30(동벽)
```

### 0-2. 시나리오 출발점·끝점·경로
각 시나리오는 **서로 다른 init/goal**로 **다른 실패모드**를 겨냥(설계: [P5_SCENARIOS.md](./P5_SCENARIOS.md)).
init(스폰=AMCL 초기포즈)와 goal(경로 끝)은 기둥 좌표를 피해 근처 개활점으로 오프셋함.

| 시나리오 | 성격 | 출발점 init (x,y) | 끝점 goal (x,y) | 경로 요지 (차선 통과) |
|---------|------|-------------------|-----------------|----------------------|
| **C1** | 대각 횡단 | (-24, -24) SW구석 | (24, 24) NE구석 | 서벽→**core(0,0)**→동벽→NE. 코어 정면 관통 |
| **C2** | 오프셋 슬라이스 | (-24, 8) 서쪽 | (24, -8) 동쪽 | (-6,8)→(6,-8)→동쪽. 코어를 비스듬히 관통 |
| **C3** | 폐루프 | (-22, -22) SW | (-22, -22) 제자리 | core 통과→NE→동벽→남벽→서벽→복귀(닫힘) |
| **C4** | core 종착 | (-24, 0) 서쪽 | **(0, 0) core 정지** | 서벽→코어로 진입해 **최악 지점서 멈춤** |
| **C5** | 지그재그 | (24, -24) SE구석 | (-18, 18) NW | 남→북 왕복하며 개활 구간 반복 관통 |

> 경로 상세 웨이포인트는 `auto_drive_demo.py` 의 `ROUTES['c1'..'c5']`, 실행 파라미터(init·yaw·maxtime)는 `run_all_baseline.sh` 참조.
> 공통: 로봇은 **오도메트리로 주행**(AMCL 추정과 무관하게 실제 경로는 정확) → AMCL의 추정만 실패한다.

회색 원=기둥(20m 격자), 주황 사각=랙, 빨간 점선원=featureless core. (`plot_scenarios.py` 로 재생성)

## 1. 지표·요소 해설 (각 수치가 무엇을 뜻하나)

### 기본 재료
| 요소 | 의미 | 읽는 법 |
|------|------|---------|
| **ground truth (GT)** | Gazebo가 아는 로봇의 **진짜 위치** (`/ground_truth`) | 정답. 플롯의 초록 궤적 |
| **estimate (est)** | AMCL이 **추정한 위치** (`/amcl_pose`) | 플롯의 빨강 궤적. 초록과 벌어질수록 나쁨 |
| **visible_beams** | 그 순간 라이다 360빔 중 물체에 맞은 **유효빔 수** | 특징이 얼마나 보이는지. 개활지에서 0에 가까움 |

### per-timestep (매 순간 값)
- **pos_err (위치오차, m)** = ‖est − GT‖. 추정이 진짜에서 몇 m 벗어났나. **가장 직관적인 정확도**.
- **yaw_err (헤딩오차, °)** = 추정 방향과 진짜 방향의 차이.
- **cov_major (공분산 장축, m·1σ)** = AMCL이 스스로 매긴 **불확실성 타원의 반장축**.
  - "내 추정이 이 반경 안에 있을 것 같다(1σ≈68%)"는 **자기 확신도**. 작을수록 확신, 클수록 "나 헷갈림".
  - RViz의 주황 타원이 이것. 개활지에서 폭발 = AMCL이 길 잃음을 **스스로 인지**.
  - (`cov_trace` = 위치 공분산 대각합, 유사 지표.)

### per-run (실행 전체 요약)
- **ATE (Absolute Trajectory Error, RMS m)** = pos_err 전체의 **제곱평균제곱근(RMS)**.
  - 궤적 전체의 **평균적 정확도** 대표값. **단계 비교의 핵심 숫자**(작을수록 좋음).
- **max_err (m)** = pos_err 최댓값. **최악 순간**에 얼마나 틀렸나.
- **final_err (m)** = 마지막 순간 pos_err. 끝에 **회복했는지/못 했는지**. (C4의 goal 오차이기도)
- **max_cov_major (m)** = 공분산 장축 최댓값. **불확실성이 얼마나 폭발**했나.
- **RPE (Relative Pose Error, 1s RMS m)** = 1초 구간의 **상대 이동 오차** RMS.
  - ATE가 "누적 절대 오차"라면 RPE는 **구간별 얼마나 튀나(점프성)**. 높으면 aliasing 점프 잦음.
- **drift_rate (m/s)** = 개활 구간(가시빔 하위 30%)에서 pos_err의 **선형 증가 기울기**. 보정 없이 밀리는 속도.
- **loop_disp (m)** = 시작 대비 est의 최종 변위. 폐루프(C3)면 이상적 0(제자리 복귀), 클수록 드리프트.
- **reconv_events (s)** = 공분산이 임계(1.0m) 초과 후 다시 하회(0.3m)까지 **재수렴에 걸린 시간들**. 비어있으면 "회복 못 함".

## 2. C1~C5 결과 분석

| 시나리오 | ATE | 최대오차 | 최종오차 | 최대공분산 | RPE(1s) | 재수렴 | drift/s |
|---------|-----|---------|---------|-----------|---------|--------|---------|
| C1 대각 횡단 | 34.2 | 60.4 | 60.1 | 17.1 | 2.24 | 없음 | 0.48 |
| C2 오프셋 슬라이스 | 23.9 | 58.7 | 32.7 | 24.6 | 11.3 | 없음 | 0.43 |
| C3 폐루프 | 32.4 | 59.1 | 35.3 | 24.9 | 4.72 | 196.8s | 0.47 |
| C4 core 종착 | **2.4** | 3.1 | 2.7 | 9.7 | 0.34 | 없음 | -0.01 |
| C5 지그재그 | 27.0 | 62.0 | 45.3 | 21.6 | 10.8 | 없음 | 0.21 |
*(단위: m, RPE·drift 제외)*

### 실패를 맵 위에서 보기 (GT vs AMCL)
맵 배경 위에 **초록=진짜 경로(GT)**, **빨강 점선=AMCL 추정**을 겹친 그림. AMCL이 얼마나 폭주하는지 직관적으로 드러난다. (`plot_scenarios.py` → `failure_overview.png`)

![C1~C5 실패 비교: GT(초록) vs AMCL(빨강)](img/failure_overview.png)

- **C1·C2·C3·C5**: 초록은 깔끔한데 **빨강이 맵 전체로 폭주** — core에서 길을 잃고 반복 기둥(aliasing)에 오매칭돼 사방으로 점프.
- **C4**: 빨강이 **초록을 바짝 추종** → 위치 오차 2.4m(온건). 단, 이 경우도 공분산은 컸음(표 참조) = "위치는 우연히 맞아도 확신은 없음".

### 시나리오별 해석
- **C1 (대각 횡단)** — 시작엔 pos_err≈0(특징 풍부). core 진입 후 계단식으로 60m까지 발산, **최종까지 60m 로스트**(회복 실패). 반대편 특징을 봐도 못 돌아옴 → aliasing.
- **C2 (오프셋 슬라이스)** — ATE는 C1보다 낮으나 **RPE 11.3로 최고** = 프레임간 점프가 극심(추정이 계속 튐). 공분산도 24.6m로 큼.
- **C3 (폐루프)** — 유일하게 **재수렴 1회(196.8s)**: 벽 근처로 복귀할 때 잠깐 확신 회복 → 다시 발산. loop_disp 54m = 제자리로 안 돌아옴(누적 드리프트 큼).
- **C4 (core 종착·정지)** — **예외적으로 온건**(ATE 2.4m). 짧게 다녀와 aliasing 점프가 안 생김. **하지만 공분산 9.7m** = "평균은 우연히 가까워도 AMCL은 확신 없음". → position만 보면 속을 수 있고, **공분산까지 봐야 실패가 드러난다**는 교훈.
- **C5 (지그재그)** — RPE 10.8, 최대오차 62m. 관측 단절을 반복하며 계속 튐.

### 시나리오 관통 패턴
1. **개활지 체류 시간 ↔ 실패 크기**: 오래 지날수록(C1·C2·C3·C5) catastrophic, 짧으면(C4) 온건.
2. **회복 불가**: 재수렴이 거의 없음(C3만 1회) → 한 번 잃으면 **반복 기둥(aliasing) 탓에 못 돌아옴**.
3. **공분산은 정직하다**: 모든 시나리오에서 실패 시 공분산이 크게(9.7~24.9m) 뜸 → AMCL 스스로 "못 믿겠다"고 신고. C4처럼 position이 좋아 보여도 공분산이 실패를 폭로.

## 3. 왜 이렇게 실패하나 (메커니즘)

1. **개활지 = 관측 부재**: core에서 라이다 유효빔이 0 → AMCL이 스캔-지도 매칭으로 **보정할 근거가 없음**. 파티클이 모션모델 노이즈로 퍼짐 → 공분산 폭발.
2. **Perceptual aliasing = 회복 불가**: 기둥이 20m 격자로 **전부 똑같이 생김**. 길 잃은 AMCL이 기둥을 봐도 **엉뚱한 기둥에 매칭** → 위치가 20m 단위로 점프, 영영 못 돌아옴. (RPE·계단식 오차가 이것.)
3. **공분산의 역할**: AMCL은 이 상황을 확신도(공분산)로 정직하게 표현 → "불확실성 폭발"이 핵심 실패 신호.

→ 이것이 프로젝트가 처음부터 제기한 문제 그 자체. **feature가 없거나 반복되면 스캔 기반 로컬라이제이션(AMCL)은 무력**하다.

## 4. 한계 & 다음 비교의 의미

**한계(해석 시 유의)**
- **단일 run**: 파티클 필터는 확률적이라 실행마다 편차 큼(같은 C1도 회복/미회복 갈림). 정식 결론엔 **반복 평균 필요**(P6).
- **odom 노이즈 없음**: 현재 실패는 *feature 부재 + aliasing* 주도. 바퀴 미끄러짐 드리프트는 미반영(P4, 선택).
- 절대 수치(60m 등)는 이 특정 환경·설정에서의 값. 중요한 건 **상대 비교**(단계 간).

**다음 단계가 개선할 지점 (제안 방식이 줄일 3값)**
| 값 | baseline 현재 | 제안 방식 기대 |
|----|--------------|---------------|
| 보정 간 잔여 드리프트 | drift 0.2~0.5 m/s, 계속 누적 | EKF(M5)로 완화, 반사판(M6)으로 주기 리셋 |
| 최대 불확실성(공분산) | 9.7~24.9m 폭발 | 반사판 절대관측으로 억제 |
| 재수렴 latency | 거의 ∞(회복 불가) | 고유 반사판으로 즉시 회복 |

**핵심**: baseline은 "AMCL이 개활 창고에서 처참히 실패"를 정량 확립했다. M5(EKF)·M6(반사판)는 **동일 시나리오·지표**로 돌려 이 세 값이 얼마나 개선되는지 보이면 프로젝트의 결론이 완성된다.
