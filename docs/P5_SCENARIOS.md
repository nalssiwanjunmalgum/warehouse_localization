# P5 — 검증 시나리오 & 지표 설계

> 로드맵상 **P5(검증 시나리오) + M4(지표 파이프라인)**에 해당. 2026-07-04 설계, 맵(P2) 완료로 착수 가능.
> 좌표는 모두 world 프레임(외벽 ±30, 기둥 ±20 격자, 중앙 featureless core).

## 목적
제안 방식(**EKF 융합 + 반사판 랜드마크**, M5/M6)이 **AMCL 베이스라인**보다 개활지에서 우수함을 보인다.
5개 케이스의 **init/goal을 전부 다르게** 하여, 각기 **다른 실패 모드**를 겨냥한다.

## 5개 검증 케이스

| ID | 이름 | init → goal | 경로 성격 | 겨냥하는 실패 모드 |
|----|------|-------------|-----------|-------------------|
| **C1** | 대각 횡단 | (-24,-24) → (24,24) | core 정면 관통 | 지속 발산 (drift) |
| **C2** | 오프셋 측면 | (-24, 8) → (24,-8) | core 슬라이스 | 간헐 관측 |
| **C3** | 폐루프 | (-20,-20) → (-20,-20) | 출발=도착 루프 | 루프 클로저 오차로 누적 드리프트 대비 |
| **C4** | core 종착 | (-20, 0) → (0, 0) | 최악 지점 정지 | 목표점 정지 정확도 |
| **C5** | 지그재그 | (24,-24) → (-20,20) | 관측 단절 반복 | 재수렴 시간 |

## 추적 지표 (2층 구조)

### ① per-timestep (매 시점)
- GT pose(정답) vs 추정 pose
- 위치 오차 / 헤딩 오차
- cross-track / along-track 오차
- AMCL 공분산 (확신도 → 개활지에서 폭발)
- 가시 특징 수 (스캔 유효빔·기둥/반사판 관측 수)
- 보정 이벤트 발생 + 점프 크기

### ② per-run (실행 단위)
- **ATE** (절대 궤적 오차), **RPE** (상대 포즈 오차)
- 최대 오차
- 루프 클로저 오차 (C3)
- goal 도달 오차 (C4)
- 재수렴 시간 (C5)
- 보정 간격 & 보정 점프
- 성공률

## "개선 여지" 강조 3값
제안 방식의 이점을 부각하는 핵심 수치:
1. **보정 간 잔여 드리프트** — 리플렉터 보정 사이에 얼마나 덜 밀리는가
2. **보정 점프 크기** — 절대 보정 시 위치가 튀는 정도
3. **재수렴 latency** — 관측 단절 후 다시 수렴하는 데 걸리는 시간
