# P3 — 센서·랜드마크 설계 (M6 선행)

> 3단계 비교의 **3단계(해결=랜드마크 절대보정, M6)** 를 위한 설계.
> 반사판을 **어떻게 감지하고(센서), 몇 개를 어디에 두고(배치), 어떻게 식별·보정하나(알고리즘)** 를 정한다.
> 관련: CLAUDE.md §3-B·§8-③ · baseline [M3_ANALYSIS](./M3_ANALYSIS.md) · 완화 [M5_ANALYSIS](./M5_ANALYSIS.md)

---

## 0. 왜 랜드마크인가 (M3·M5 가 남긴 숙제)

- **M3**: 개활지에서 AMCL 은 perceptual aliasing 으로 절대 위치를 못 잡음(cov 폭발, ATE 24~34m).
- **M5**: EKF(오도+IMU)는 추측항법 드리프트만 완화, **aliasing 은 못 고침**(cov 여전히 큼). = 완화≠해결.
- **남은 문제 = 절대 위치의 부재.** 개활 core 에서 로봇에게 "너 여기야"라고 말해줄 **고유하고 절대적인 관측**이
  필요하다. → 알려진 좌표에 **반사판(랜드마크)** 을 심고, 3개 이상 보이면 **삼각측량으로 절대 위치를 확정**한다.
  이것이 실제 창고 AGV(SICK·Kollmorgen 등)의 표준 방식.

---

## 1. 센서 — intensity 검출을 시도했으나 막힘 → 기하(geometric) 검출로 결정

원래 계획(CLAUDE.md)은 반사판에 `<laser_retro>` 를 부여해 **LiDAR intensity 가 튀는 지점**을 임계값으로
검출하는 것이었다. 그러나 **이 시뮬 환경에서 intensity 경로가 막혀** 있음을 실측으로 확인:

| 방식 | 결과 | 원인 |
|------|------|------|
| CPU `ray` 센서 + `laser_retro` | intensities[] 는 나오나 **전부 0** | Gazebo Classic CPU 레이는 retro 반사율 미시뮬(문서화된 한계) |
| `gpu_ray` 센서 | 헤드리스에서 **스캔 자체가 안 나옴** | GPU 오프스크린 렌더 컨텍스트 필요 → 헤드리스 배치 평가에 부적합(사실상 gzclient 필요, 느리고 불안정) |

→ **결정: intensity 대신 기하 검출.** 실제 CPU LiDAR range 데이터로 반사판을 **모양(얇은 기둥)으로 검출**하고,
보이는 반사판들의 **상대 배치(성좌)로 식별**한다. 헤드리스에서 견고·결정적으로 동작한다.
(IMU 노이즈는 이미 모델에 설정됨: 각속도 σ=2e-4, 가속도 σ=1.7e-2 — M5 EKF 가 사용 중.)

> **얇게 만드는 이유**: 반사판을 **가는 폴(지름 0.16m)** 로 만들면, 기둥(지름 0.4m)·랙(박스)과
> **클러스터 각폭**으로 구별된다. 게다가 개활 core 에선 기둥이 12m 밖이라 **LiDAR 에 반사판만 잡힌다**
> → 그 구간의 검출은 사실상 자명(어떤 리턴이든 반사판).

## 2. 반사판 물리 설계

| 항목 | 값 | 이유 |
|------|-----|------|
| 형상 | 원기둥 폴 | 2D LiDAR 에 방향무관 일정 단면 |
| 반지름 | **0.08m** (지름 0.16m) | 기둥(0.2m)·랙보다 얇음 → 클러스터 폭으로 구별. 12m 에서 ~1빔, 근거리 다수빔 |
| 높이 | 1.2m (z 0~1.2) | LiDAR 평면(z≈0.18m)을 확실히 가로지름 |
| 재질 | 노란색 visual | 시각 식별용(검출엔 무관, intensity 안 씀) |
| static | true | 고정 랜드마크 |

배치는 **오버레이 월드**(`warehouse_reflectors.world` 또는 베이스 월드 include + 반사판 추가)로만 넣어,
베이스 월드/occupancy 맵은 **그대로 유지**(공정 비교, §6).

## 3. 배치 & 커버리지 (16개, 불규칙 배치)

**요구조건 2가지 (상충):**
1. **커버리지** — 로봇이 지나는 개활 구역 어디서든 12m 내 반사판 **≥3개**(삼각측량 성립).
2. **고유성(비-aliasing)** — 반사판 배치가 규칙적이면 성좌가 반복돼 식별이 다시 aliasing 에 빠짐 →
   **불규칙 배치**로 국소 성좌를 고유하게.

**설계**: 기둥 격자(20m)와 5m 오프셋된 레인 중앙 `{-15,-5,5,15}²` 기본격자(16점)에 **결정적 지터 ±3.2m**
(기둥 0.9m·랙 1.5m 회피). 지터가 규칙성을 깨 성좌를 고유화하면서 커버리지를 유지.

**검증(계산):**
- 중앙 개활 ±15 구역: 12m 내 반사판 **중앙값 4개, ≥3 만족 98.1%** (미달 2%는 가장자리 = 거긴 기둥·벽 특징 있음).
- core(0,0): **4개**(거리 5.4·6.5·7.2·9.6m, 방위 분산 양호 → 삼각측량 기하 좋음).
- 성좌 고유성: 규칙격자 대비 혼동쌍 188→**24** 로 급감(지터 효과).

**반사판 좌표 (m, 랜드마크 DB = 이하 M6 `landmarks.yaml`):**

| id | x | y | id | x | y |
|----|------|------|----|------|------|
| R00 | -14.67 | -13.67 | R08 | 1.95 | -14.62 |
| R01 | -16.34 | -4.93 | R09 | 3.46 | -5.54 |
| R02 | -12.49 | 7.54 | R10 | 3.61 | 6.24 |
| R03 | -17.40 | 13.13 | R11 | 4.62 | 12.80 |
| R04 | -7.87 | -15.38 | R12 | 15.29 | -13.21 |
| R05 | -8.01 | -5.28 | R13 | 13.76 | -6.78 |
| R06 | -4.05 | 3.58 | R14 | 14.28 | 7.79 |
| R07 | -3.87 | 15.58 | R15 | 18.05 | 16.10 |

> 개수(밀도)는 **개선 여지** 축이다. 16개는 중앙 개활을 ≥3 로 덮는 최소에 가까운 값. 부족 구간이 나오면 증설.

**배치 & 커버리지 시각화** (`plot_reflectors.py`): 왼쪽=반사판(파랑 ▲) 흩뿌림 + 기둥(회색)·랙(주황),
오른쪽=12m 내 가시 반사판 수 히트맵(**빨간 선 = ≥3, 삼각측량 가능 영역**). 중앙 개활이 잘 덮인다.

## 4. 랜드마크 좌표 DB (`config/landmarks.yaml`, M6)

삼각측량 노드가 참조하는 **"내 방식의 지도"**. occupancy 격자(AMCL 지도)와 **별개 파일**.
```yaml
# id, x, y  (map 프레임, m). 물리 반사판(월드)과 반드시 일치.
landmarks:
  - {id: 0, x: -14.67, y: -13.67}
  - {id: 1, x: -16.34, y: -4.93}
  # ... R00~R15
frame_id: map
reflector_radius: 0.08
```

## 5. 검출·식별·보정 알고리즘 (M6 구현 스케치)

```
/scan(360빔) ──► [검출] 연속 리턴 클러스터링 → 각폭으로 지름 추정
                        → 지름<0.16m 이며 고립된 클러스터 = 반사판 후보
                        → 각 후보의 (range, bearing) [로봇좌표]
                 [식별] 후보들의 상대 성좌(쌍거리·각) ↔ landmarks.yaml 매칭
                        (현재 추정 pose 로 게이팅 + 성좌 일관성 검증 → 오연관 방지)
                 [보정] 매칭 ≥3개 → 최소자승으로 map 프레임 절대 pose 해
                        → map→odom 보정 발행(AMCL 대체/보완). cov 급감.
```
- **검출**: core 에선 반사판만 보이므로 자명. 전이구간(기둥도 보임)은 **폭 필터**로 기둥 배제.
- **식별(성좌)**: 관측 반사판들의 **상대 기하는 pose 불변** → 나쁜 prior(aliasing)에도 견고. 시작은
  알려진 초기 pose 에서 부트스트랩, 이후 **연속 보정**으로 estimate 를 진실 근처로 유지 → 연관 안정.
- **보정**: 3개 이상이면 2D 절대 pose(x,y,θ) 과결정 → 최소자승. → `map→odom` 갱신 = **절대 위치 확정**.
- EKF(M5)는 **유지**: 반사판 보정 사이 구간의 추측항법 안정화.

## 6. 공정성 — occupancy 맵엔 반사판을 넣지 않는다

3단계 비교가 공정하려면 **baseline·EKF 가 반사판 덕을 보면 안 된다**.
- 물리 반사판은 **오버레이 월드**(M6 전용)로만 추가 → baseline/M5 월드는 그대로.
- AMCL 이 쓰는 **occupancy 격자(warehouse_map)에는 반사판을 그리지 않는다**(맵은 반사판 없이 만든 것 유지).
  → AMCL 은 여전히 개활지에서 특징을 못 봄. 반사판은 **오직 M6 삼각측량 노드만** 활용.
